Welcome to ng-catering!

This app is a PoC for a catering menu builder. You create printable menus from a list of dishes, which are composed from a list of ingredients. The ingredients can be flagged as containing meat/seafood or gluten, or if they're spicy. Flagged ingredients will be displayed with a corresponding icon that flows through into the dishes and menus they are apart of. This demo comes with some hardcoded data which you're free to manipulate during the session, but changes will not persist between reloads or multiple browser tabs.

{{ingredientNum}} Ingredients

Ingredients form dishes and can have properties such as being spicy, or containing meat/seafood or gluten.

About

This app was created using Angular (using TypeScript) and the Clarity Design UI Framework. As far as modern JS frameworks go, Angular would be my pick (especially over React). Mostly because it feels like a "complete package" that isn't clunky to work with, or dogmatic about its patterns. Clarity Design seems clean and slick, although laying out forms with more complex control combinations seemed a bit awkward. I don't like some of its default styling, which is easy enough to fix by overriding the classes. I expect I'll use it again, though.

For guts, it's a pretty typical SPA. I was just interested in client-side for this, so the data isn't persistent. I did write the services such that they used observables (even though they contain static data) because that's nearer to a real-world setup. Some polish could be added to the UI, and some UX improvements could be added, but as-is it's fine for a PoC.
